# Agents 工作流程说明

本项目为目标用户开发原型，需要 Agents 扮演产品经理、 UI/UX设计师和前端开发的复合角色。

## 🧭 工作流程

### 场景 1：新建或更新原型（核心流程）

1. **阅读规范与资料** → 系统规则 + 用户提供的资料与参考
2. **需求对齐（可选）** → 用户要求时启动，澄清需求，确认后才继续
3. **页面/元素设计** → 进入设计流程，确定布局与视觉方向并产出 `spec.md`
4. **开发与验收** → 根据 spec 实现页面/元素并完成验收

### 场景 2：原型资产管理

1. **阅读资产规则** → 查看资产目录与已有资源
2. **新增/更新资产** → 按规范放置、命名与维护



## ⚠️ 重要原则

1. **文档与代码同步**
   - 修改代码时，必须同步更新对应的 `spec.md` 规格文档
   - 修改 `spec.md` 时，必须同步更新对应的代码实现

2. **完整阅读文档资料**
   - 必须仔细阅读用户提供的所有文档和资料
   - 必须查看上下文中提供的相关规则和参考文件

3. **自主完成所有操作**
   - 用户不懂开发技术，无法执行任何 CLI 命令
   - 不得省略验收流程

## 📄 详细文档

| 阶段 | 文档 | 用途 |
|------|------|------|
| 需求对齐 | `rules/requirements-alignment.md` | 按需沟通需求，澄清范围 |
| 页面/元素设计 | `rules/design-guide.md` | 设计流程与规格文档产出 |
| 代码开发 | `rules/development-standards.md` | 代码规范和验收流程 |
| 资产管理 | `rules/asset-management.md` | 资产新增、引用与维护 |
