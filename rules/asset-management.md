# 原型资产管理规则

本文档用于规范原型资产的新增、引用与维护，保持项目资产清晰可复用。

## 🎯 目标

- 资产可追溯、可复用、可维护
- 避免重复或不可控的素材堆积
- 确保页面/元素引用路径稳定

## 📁 资产目录约定

- `/assets/docs/`：规范文档、设计说明、参考资料
- `/assets/libraries/`：前端库与组件库文档
- `/assets/database/`：页面可直接消费的数据表 JSON
- `/assets/templates/`：模板与通用草稿
- `/assets/themes/`：主题配置与主题资源（如需构建，可放在 `src` 内对应目录）

## 🔗 相关规则

- 主题生成规范：`rules/theme-generation-guide.md`

## ✅ 管理规则（简要）

- 新增资产前先检查是否已有可复用资源，避免重复
- 新增资产时使用清晰可读的命名（可用中文），并保持同类资产命名风格一致
- 主题资产可在 `src` 内对应目录维护以支持构建，但仍视为原型资产并纳入统一管理
- 未经用户确认，不删除或覆盖已有资产
- 数据资产需遵循 `/assets/database/README.md` 的格式要求
