# 远程控制机器人可视化大屏设计规格说明书

## 1. 业务与功能

### 1.1 页面定位
本页面是机器人远程操作与监控中心，旨在为操作员提供一个直观、高效的界面，用于实时监控机器人状态、接收传感器数据、发送控制指令以及查看任务执行情况。

### 1.2 功能需求
- **实时视频监控**：展示机器人摄像头回传的实时视频流。
- **机器人状态显示**：以仪表盘形式展示机器人的关键状态参数（如：电池电量、连接信号、电机温度、当前速度）。
- **环境数据显示**：展示机器人传感器收集的环境数据（如：温度、湿度、空气质量）。
- **手动控制面板**：提供方向控制（前进、后退、左转、右转）和功能操作（如：机械臂抓取、摄像头云台转动）的按钮。
- **任务日志与状态**：显示机器人当前执行的任务、历史操作日志和系统警报。
- **3D姿态可视化**：实时展示机器人的三维姿态模型。

## 2. 内容与信息架构

### 2.1 信息分区
- **主视觉区**：视频监控 + 3D姿态模型。
- **状态监控区**：机器人核心状态仪表盘。
- **环境感知区**：环境数据图表。
- **交互控制区**：手动控制按钮和任务指令面板。
- **信息流区**：任务日志与系统警报。

### 2.2 数据来源
- **视频流**：来自机器人摄像头的实时推流（原型阶段使用占位视频或图片）。
- **状态与环境数据**：通过 WebSocket 或 API 从机器人后端获取（原型阶段使用模拟数据）。
- **控制指令**：通过 API 或 WebSocket 发送至机器人后端。

## 3. 布局与结构

### 3.1 整体布局
- **布局模式**：采用灵活的多区域网格布局，确保在不同分辨率的大屏上都能清晰展示。
- **背景**：深灰色或深蓝色调，营造专业、科技感的氛围。
- **栅格系统**：12列栅格系统。

### 3.2 区域划分
- **左侧区域 (3/12)**：
  - **顶部**：机器人状态仪表盘（电池、信号等）。
  - **底部**：手动控制面板。
- **中间区域 (6/12)**：
  - **主区域**：实时视频流。
  - **右下角浮动窗口**：3D姿态可视化。
- **右侧区域 (3/12)**：
  - **顶部**：环境数据显示图表。
  - **底部**：任务日志与警报列表。

## 4. 视觉与交互规范

### 4.1 视觉风格
- **主色调**：科技蓝 (`#00A4FF`)、深空灰 (`#2C3E50`)。
- **辅助色**：警告红 (`#E74C3C`)、正常绿 (`#2ECC71`)、信息黄 (`#F1C40F`)。
- **字体**：使用无衬线字体（如 `Roboto`, `Helvetica`），保证清晰易读。
- **图标**：使用 `lucide-react` 图标库，风格统一、简洁明了。

### 4.2 交互设计
- **控制按钮**：具有明确的点击、悬停和禁用状态。支持键盘快捷键操作（如方向键）。
- **数据刷新**：状态和环境数据应实时刷新，并有加载状态提示。
- **警报通知**：出现高级别警报时，在界面显著位置弹出提示，并伴有声音提醒。
- **图表交互**：支持缩放、平移和数据点高亮显示。

## 5. 原型实现要点
- **数据模拟**：所有动态数据均使用 `useState` 和 `useEffect` 结合 `setTimeout` 或 `setInterval` 进行模拟。
- **视频占位**：使用 `picsum.photos` 或静态图片作为视频流占位符。
- **3D模型**：使用 `ECharts` 的 `series-surface` 或一个简单的 `div` 元素作为3D姿态的占位符。
- **组件化**：将各个功能区域（如状态仪表盘、控制面板）拆分为独立的React子组件，提高代码复用性和可维护性。
